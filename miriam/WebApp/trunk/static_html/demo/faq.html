General questions:

What does MIRIAM stand for?

MIRIAM is an acronym for Minimal Information Requested In the Annotation of biochemical Models.

What is MIRIAM Standard?

MIRIAM Standard is an effort to standardise the Minimal Information Requested In the Annotation of biochemical Models, so that different groups can collaborate on annotating and curating computational models in biology. The goal of the project, initiated by the BioModels.net effort is to produce a set of guidelines suitable for use with any structured format (SBML, CellML, GENESIS, CellML, ...) for computational models.

More information is available on the MIRIAM Standard page.

What are MIRIAM Resources?

As explained on the MIRIAM Standard page and just above, the community has to agree on a set of standard valid URIs. MIRIAM Resources is a set of services and resources composed of a database, powered by MySQL (MIRIAM Database), SOAP-based Web Services, using Apache Axis (MIRIAM Web Services), a library to access these Web Services, currently available in Java, but other languages will be supported soon (MIRIAM Library) and a Web application, running with Apache Tomcat (MIRIAM Web Application).

The database and the associated API (Web Services) have been developed at the European Bioinformatics Institute to provide the generation and interpretation of MIRIAM URIs.

MIRIAM Resources is available at this address: http://www.ebi.ac.uk/miriam/.

What is a MIRIAM URI?

MIRIAM URIs is a way to identify, in a stable and perennial way, elements, like proteins, genes, publications, etc. It was first designed to manage components of computational models, but the system can be used for other purposes.

As an example: urn:miriam:pubmed:16333295 is the MIRIAM URI of the publication about MIRIAM.

A MIRIAM URI is composed of two parts. The first one is the URI of the data type (PubMed, UniProt, Ensembl, ...). The second part is the identifier of a precise element within this data type (for example, in the case of a publication, this identifier will be the PMID).

Why do we need MIRIAM URIs?

The usage of MIRIAM URIs permits:

  • uniquely describe entities with a perennial and stable identifier;
  • link an entity to one or more online resources, where extra knowledge about it is available;

MIRIAM URIs are not dependant on physical addresses (like the URLs used with a web browser), that is why they are persistent.

What is a data type?

A data type is a unique, controlled description of the type of the data. For example, if the entity is a protein, the data type will be UniProt, if the entity is an enzyme, the data type will be Enzyme Nomenclature, ...

What is a resource?

A resource is a service (frequently a database accessible on-line through a Web-based interface) providing information about the elements of a data type. For each data type, several resources can exist. Some are real mirrors, but others provide a slightly different information.

What are the resolving services provided for MIRIAM URIs?

MIRIAM Resources provide Web Services in order to perform several resolution processes. There are two main different ways to use the resolution service. One way is designed to be used by curators or model creators and the other one is for software developers.

For a curator (or even a model creator), the need to put some extra knowledge in an already created (or new) model is mandatory. Whatever the format is to store the model (SBML, BioPAX, CellML, \dots), in order to obtain a MIRIAM compliant model, all the components must be linked with a MIRIAM URI. However, the curator knows the components of his model. For example, if one of them is a F-actin capping protein subunit beta, he knows its primary accession number (P47757) used by UniProt (Universal Protein Resource). Therefore, he just need to use his favourite software tool (the one he usually uses to curate or annotate models, like SBMLeditor) to generate the appropriate URI, providing these two pieces of information. Of course, this tool just need to use the method called getURI() of MIRIAM Web Services.

As an example, if a user wants to retrieve the MIRIAM URI of the protein P47757 (via UniProt), he or she will obtain: urn:miriam:uniprot:P47757.

A developer works with models already encoded, so in a digital form. If he wants to create an interface (Web-based or rich client) to display them, he needs to convert all the MIRIAM URIs into physical addresses that can be used to recover the knowledge stored in the annotations (mainly via online resources). The conversion from MIRIAM URIs to physical addresses can be done using the getDataEntries() method of MIRIAM Web Services. In the case of a Web-based interface, the developer will be able to link all the model constituents towards more information.

As an example, if a user wants to retrieve the physical locations of extra information about the entity urn:miriam:uniprot:P47757, he or she will obtain the following list:

  • http://www.ebi.uniprot.org/entry/P47757
  • http://www.pir.uniprot.org/cgi-bin/upEntry?id=P47757
  • http://us.expasy.org/uniprot/P47757

More information is available from the Web Services page.

Questions about creation/edition/update of data types:

Who maintains MIRIAM URIs and their associated data?

Data types are submitted by database creators, maintainers or users who need them.

It is important to notice that the data types are not updated automatically when their associated resources change. We regularly check the validity of the content of the database and do our best to keep it up to date. However, if you are the creator or maintainer of a data type, or just a user who notice an obsolete record in our system, you are more than welcomed to inform us. For this purpose, you can use our support form. Thank you for your help.

How to create a new data type?

Everybody can create a new data type, using the submission page.

If the user is registered, the data type will be added instantly, if not, the addition will be done after a human checking process.

How to modify/update an existing data type?

Everybody can modify an existing data type, using the browsing page. Once the details of a data type are displayed, an edition icon is available on the left of the name of the data type. That brings to a form where all the recorded data can be modified.

If the user is registered, the data type will be updated instantly, if not, the modifications will be applied after a human checking process.

Can a user delete a MIRIAM URI?

Users cannot access directly to MIRIAM URIs. These URIs only exist through a data type and data types can't be removed by users.

If a user modify a data type and remove its URI, it will not be deleted, but only become obsolete. Moreover, users can't delete a data type, but we can make a data type obsolete if it doesn't conform with our terms and conditions any more.

Questions about registered users:

Who are registered users?

Registered users are from the BioModels.net initiative.

What can unregistered users do?

Users can perform several action via the web interface:

  • access to all the data type available
  • view all the information stored about a data type
  • create new data types
  • modify existing data types
  • have an overview of the Web Services deployed
  • use the Web Services
  • export the content of the database
  • read several pieces of documentation: MIRIAM Standard, BioModels.net qualifiers or frequently asked questions
  • ...

What can registered users do?

Registered users can do the same actions as unregistered users, but without going through a checking process: all the modifications are instantly done on the database and available for all users.

Web Services:

What are the services available?

There is an overview of the Web Services available. The WSDL is also accessible here.

How to use the web services?

The easiest way to access the Web Services provided is to used the MIRIAM Library. An example of usage is also given there.

Currently only a Java version of the library is available, but it is planned to distribute a Perl module soon.

How to get the source code?

The source code of MIRIAM Library is available, under the terms of the GNU General Public License, on MIRIAM Resources and on SourceForge.

The source code of MIRIAM Web Services and MIRIAM Web Application is available, under the same license, in the SourceForge SVN repository of the project.
