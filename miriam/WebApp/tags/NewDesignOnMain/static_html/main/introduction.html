MIRIAM is an effort to standardize the Minimal Information Requested In the Annotation of biochemical Models, so that different groups can collaborate on annotating and curating computational models in biology. The goal of the project, initiated by the BioModels.net effort is to produce a set of guidelines suitable for use with any structured format for computational models.

Miriam Standard

An important part of the standard consists in the controlled annotation of model components, based on Uniform Resource Identifiers. For more information on the use of those URIs to annotate models, see the specification of SBML Level 2 Version 2. In order to enable interoperability of this annotation, the community has to agree on a set of recognised data-types. MIRIAM Resources (using a MySQL database) is an online service created to catalogue the data-types, their URIs and the corresponding physical URLs or resources, whether these are controlled vocabularies or databases.

Each entry contains a diverse set of details about the data-type: official name and synonyms, root URIs (URL and URN forms), patterns of identifiers, documentation, etc. Each data-type can be associated with several resources (or physical locations). For instance:

  • name: Gene Ontology
  • synonym: GO
  • pattern: GO:\d{7}$
  • URI: http://www.geneontology.org/
  • Resources:
    • location home: http://www.ebi.ac.uk/ego/
    • root URL: http://www.ebi.ac.uk/ego/DisplayGoTerm?selected=$1
    • location home: http://www.godatabase.org/cgi-bin/amigo/go.cgi
    • root URL: http://www.godatabase.org/cgi-bin/amigo/go.cgi?query=$1
    [...]

Note that the URIs can be take either a URL form (http://www.uniprot.org/) or a URN form (urn:lsid:uniprot.org). A URI is NOT a "real" website, but a unique string. For instance the URI representing the enzyme classification is http://www.ec-code.org/.

Miriam Resources

A freely available web site is been developed, using an Apache Tomcat server to provide an interface for the database. Users will be able to perform queries such as retrieving valid physical locations (URLs) corresponding to a given URI (whether a generic data-type or a precise piece of knowledge), retrieving all the information stored about a data-type (such as its name, its synonyms, links to some documentation, etc.) and immediately get the results in a dynamic way, using Asynchronous JavaScript and XML (AJAX).

Moreover, a programmatic access through web services (based on Apache Axis and SOAP messages) will be released soon. This API will permit not only to resolved model annotations, but also to generate the correct URIs based on resource name and accession numbers.
